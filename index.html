Welcome to WebTopoTools, an exploratory tool for geomorphologist aiming to provide easy access to state-of-the-art research algorithms. This is an alpha version of the tool, currently in development. The tool is full client-side, meaning that everything is happening on your machine: no data transfer, gathering or costly server management. It is powered by webassembly, a technology enabling compilation of fast lower-level codes (c/c++/...) to something executable by web browser. WebTopoTools relies on this technology to cross custom c++ codes with pyodide (a webassembly CPython) and loam-GDAL (geospatial data processing) and deliver fast routines dedicated to geomorphological analysis.
The current workflow is:

  1. Initialisation phase (it takes few seconds to fetch and load the codes)
  2. Load a tif Digital Elevation Model (preferentially in projected coordinates)
  3. Select analysis
  4. Tune parameters if any
  5. Run the analysis
  6. Back to 3 until you are done

Few things to be aware of: this tool is very much experimental and might crash or softlock unexpectedly. If you refresh the page multiple times, reopen a new tab to clear memory. NaN and no data are not managed well (yet). There are technical limitations making the use of large files (difficult to give an approximation, but let's say >100Mb) difficult but this technology is evolving FAST. The software has been tested on Chrome and Firefox, MS Edge should be fine as based on chromium but Safari can be randomly unstable and buggy for unknown reasons (don't get me started on that). Finally, I am a novice in front-end, so excuse poor graphics.
